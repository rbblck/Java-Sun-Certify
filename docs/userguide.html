Bodgitt and Scarper, LLC

Contractor Booking System

Welcome to the the Bodgitt and Scarper Contractor Booking System.

Here you can learn about this application and how to use it.

The Bodgitt and Scarper Contractor Booking System application will help the Customer Service Representative (CRS) take requests from home owners for a type of service and offer the homeowner one or more Contractors that can provide the required services. Once the Contractor with the desired services has been found, the CSR can then book the Contractor with the Customers eight digit Customer number.

Below is a step by step guide to searching and booking Contractors to Customers.

  1. Installation and Start Up.
  2. Select the Database Location at start up.
  3. Search for Contractors.
  4. Book a Contractor to a Customer.

1. Installation and Start Up.

There are two ways to run the Bodgitt and Scarper booking system application:

  1. In Stand Alone mode.
  2. In Network Mode.

For both modes please change your current working directory to the directory you have installed the application before executing commands.

Specific instructions to start the application in both modes follow:

  1. In Stand Alone Mode In stand-alone mode, you will only need to start one JVM to run the application as the server is not used in this mode.

    Once in the installed directory, runs the following command (using the “alone” flag):

        java -jar runme.jar alone

    The Application will now present a frame allowing you to state the location of the database. For more information on this see the ”Select the Database Location at start up” section 2.

  2. In Network Mode

     In network mode, you will need to start the server in one JVM and the client in another JVM. The JVM’s can be on the same computer or different computers connected through a network.

    First you will need to start the server in the first JVM. To do this, once in the installed directory, run the following command (using the “server” flag):

        java -jar runme.jar server

    The server will now presents a frame allowing you to state the location of the database and the port that you would like the server to listen on. For more information on this see the ”Select the Database Location at start up” section 2.

    Second you will need to start the network client in the second JVM. To do this, once in the installed directory, run the following command (using no flag):

        java -jar runme.jar

    The client will now presents a frame allowing you to state the location of the server which in turn has the location of the database and the port that you would like the client to connect on. For more information on this see the ”Select the Database Location at start up” section 1.

2. Select the Database Location at start up

When the application first starts up you are presented with a configuration window. The configuration window will require you to confirm certain properties that the application needs to read the database, depending on which mode you have started the application.

In server mode "server":

First, you will need to select the location of the database file. You can do this by using the the browse button "..." to the right of the Database Location field.
Note: You must have selected a database file location to be able to start the server.

Second, you will need to enter the port number ranging from 1025 and 49152 that you would like the server to listen on.
Note: You must enter the port number before you can start the server.

Once the above has been entered then you can start the server. The server will start with the entered file location and port number.

To start the server click the "Start Server" Button.
Note: You now can not change any of the information you have entered. If you whish to change the information you have entered, you must click the "Stop Server" button, re-enter the information and click "Start Server" again to restart the server.

In Network Client mode:

First, you will need to enter the location of the database by entering the address of the server. Either an IP address or "localhost"

Second, you will then have to enter the port number that the server is listening on.

Once the above has been entered you can connect to the server by clicking the "Confirm" button and the Network Client will start.

Note: If you do not enter the correct server address and port number, the connection will fail and you will have to restart the client application again.

In Stand Alone mode:

First, you will need to select the location of the database file. You can do this by using the the browse button "..." to the right of the Database Location field.

Once the above information has been entered correctly the application will start by clicking the "Confirm" button.
Note: If the file is not found or it is the wrong type of file the application will shut down and you will have to restart the stand alone application again.

3. Search for Contractors

The search functions are the same in either the Network Client or the Stand Alone Application modes.

To search for a list of contractors:

You can search the data for all records by not entering any search criteria, or for records where the name and/or location fields exactly match your entries. If any if the fields are left empty the search will match the other non-empty field exactly disregarding the empty field and returning all records only matching the non-empty field exactly.

Once you have entered the search criteria you have chosen click the search button and the search results will be displayed in a display table at the centre of the window.

4. Book a Contractor to a Customer:

The contractor booking functions are the same in either the Network Client or the Stand Alone Application modes.

To book a contractor with a customer number:

You must first search for a list of contractors using the search operation in step 2 so a display table is shown with a list of Contractors to choose from.

Once you have some or all contractors displayed in the display table you can select one from the table by clicking it. You then click the "Book" button below the display table to start the booking process.
Note: If a contractor has been booked while you are in the process of booking a message will be shown and the list will refresh showing the current status of all your searched contractors. If this occurs you will have to book another contractor or abort the operation.

After you have clicked the "Book" button, another window will open and with the selected contractor details, check the details, enter the customer eight digit number and click the "Book" button to book the contractor.
Note: The system will not accept anything but an eight digit customer number.

Once the contractor is booked, a conformation message will appear indicating the booking operation has been completed successfully. Also the display table will be updated to reflect your booking.
